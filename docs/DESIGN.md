# DESIGN

이 문서는 `art-market-place`의 화면과 UX 방향을 정리한 초안이다.

## 제품 톤

- 예술 작품과 콘텐츠를 다루는 서비스이므로 차분하고 읽기 쉬운 구성을 우선한다.
- 블로그 화면은 콘텐츠 탐색성과 본문 가독성이 중요하다.
- 상품 화면은 이미지, 제목, 상태, 태그, 판매자 정보를 빠르게 파악할 수 있어야 한다.

## 레이아웃 원칙

- 첫 화면에서 사용자가 현재 위치와 다음 행동을 알 수 있어야 한다.
- 목록 화면은 검색, 카테고리, 페이지 이동을 자연스럽게 제공한다.
- 상세 화면은 본문, 댓글, 좋아요 같은 상호작용을 본문 흐름을 방해하지 않게 배치한다.
- 모바일에서는 헤더와 주요 액션이 겹치지 않도록 충분한 간격을 둔다.

## UI 컴포넌트

- 버튼, 입력, 카드, 다이얼로그, 드롭다운은 기존 `components/ui/` 컴포넌트를 사용한다.
- 반복되는 항목은 카드 또는 리스트 아이템으로 통일한다.
- 아이콘은 `lucide-react`를 우선 사용한다.
- 텍스트 버튼보다 의미가 명확한 아이콘+텍스트 버튼을 선호한다.

## 색상과 타이포그래피

- 단일 색상 계열만으로 화면을 구성하지 않는다.
- 본문 가독성을 위해 충분한 대비를 유지한다.
- 제목, 본문, 보조 텍스트의 크기 위계를 명확히 한다.
- 긴 한국어 문장이 버튼이나 카드 안에서 넘치지 않도록 줄바꿈과 최대 너비를 고려한다.

## 블로그 UX

- 게시글 목록은 제목, 카테고리, 작성자, 조회수, 좋아요, 작성일을 빠르게 스캔할 수 있어야 한다.
- 상세 페이지는 본문 읽기를 우선하고, 댓글/좋아요는 보조 흐름으로 둔다.
- 카테고리 트리는 현재 선택 상태와 하위 구조를 명확히 보여준다.

## 인증 UX

- Google OAuth 흐름은 사용자가 “외부 인증으로 이동한다”는 점을 명확히 알 수 있게 한다.
- 인증 만료 메시지는 짧고 직접적으로 표시한다.
- 로그인 후 사용자 메뉴에서 상태를 확인하고 로그아웃할 수 있어야 한다.

## 상품 UX

- 이미지 업로드는 사용자가 하나의 등록 화면에서 시작하되 내부적으로 Presigned URL 발급, 클라이언트 업로드, 상품 등록 순서를 따른다.
- 업로드 실패/복구 상태는 사용자가 다시 시도할 수 있게 설명한다.
- 상품 등록 화면은 필수 입력과 선택 입력을 명확히 구분한다.

## 접근성

- 클릭 가능한 요소는 키보드 포커스와 hover 상태를 가진다.
- 폼 오류는 입력 필드 가까이에 표시한다.
- 색상만으로 상태를 전달하지 않는다.
